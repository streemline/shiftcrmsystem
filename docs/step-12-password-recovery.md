# Отчет: Шаг 12 — Экран восстановления пароля

## Что было сделано

1. **Создана страница ResetPassword** (`src/pages/ResetPassword.jsx`):
   - Обрабатывает переход по ссылке сброса из email
   - Проверяет наличие валидной сессии (Supabase автоматически устанавливает её из токена в URL)
   - Слушает событие `PASSWORD_RECOVERY` через `onAuthStateChange`
   - Форма ввода нового пароля с подтверждением
   - Валидация: минимум 6 символов, совпадение паролей
   - `supabase.auth.updateUser({ password })` для сохранения
   - Автоматический редирект на главную после успеха
   - Обработка истекших/невалидных ссылок

2. **Обновлен App.jsx**:
   - Маршрут `/reset-password` добавлен ВНЕ `AuthenticatedApp`, чтобы работать без аутентификации
   - Вложенные Routes для корректного роутинга

## Полный flow восстановления пароля

1. Пользователь на LoginPage нажимает "Забыли пароль?"
2. Вводит email -> `resetPasswordForEmail()` отправляет ссылку
3. Пользователь переходит по ссылке из email -> попадает на `/reset-password`
4. Supabase обрабатывает токен из URL, устанавливает сессию
5. Пользователь вводит новый пароль -> `updateUser()` сохраняет
6. Автоматический редирект на главную (уже аутентифицирован)

## Результат
Полный цикл восстановления пароля работает от начала до конца.
